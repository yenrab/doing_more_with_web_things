Remediation

The history of the class has shown, as indicated in the hours per week table, that those who are less fluent in Java have to work harder and longer to achieve the grade they would like compared to those that are more fluent. If you feel a possible need for remediation, you can do this by retaking Object Oriented Programming I or through self study. I’ve written a book to help out with self study. It is available on Apple’s iTunes book store and as a PDF. The sample code, PDF version of the book, and other resources are part of the zip file you can download from the Doing Stuff with Java gitHub repository.

Manage Your Time

Your time and the time of your teammates is valuable. You will want to manage your time well. There is no suggested order for learning the technical topics in this course. You should put together a personal and team schedule that allows you to get to a fluency level of 'disagree' in the course rubric by the end of week 8.

Because the assessment items in the 'Code Topics Group' and the 'UML Diagrams and Tests Group' total to more than eight, you will not be able to plan to learn one topic a week. Plan ahead.

While there is no required or suggested order for you and your team to learn the topics, here is an order that other students have suggested might be good for you. Remember, this doesn't mean one topic per week. All of these are discussed in the book, but you may also need to go online to find more resources after you've read the book or look in the gitHub download for examples of the UML diagrams and documents. Online resources are not helpful with learning the UML for this course.

  1. Java Collections
  2. Threads, Executors, and Runnables
  3. Application Controller Pattern
  4. MVC
  5. JavaBeans
  6. Hibernate
  7. QCJSON and JSON
  8. Android HttpUrlConnection
  9. Servlets
  10. JUnit Tests
  11. System Level Tests
  12. Use Case Diagrams
  13. Use Case Documents
  14. State Diagrams
  15. Sequence Diagrams

Hmmm...8 weeks and 15 topics. Plan well.   :)
